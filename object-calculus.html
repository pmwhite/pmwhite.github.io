Object calculus.

Object calculus is an alternative to lambda calculus as a minimal foundation of computation. The one construct in the language is the "object", which encodes both data and computation. You can play around with it in the sandbox editor below.

Syntax

An object is a series of "equations", which are pairs of "expressions". An expression is a name, object, fixpoint, field access, merge, or resolve. The following examples illustrate these cases:

The empty object.

{}

The abstract name

name

Equations of two names.

{ x = x }
{ a = b }

Equations between variable and object.

{ x = {} }
{ x = { x = { x = { x = {} } } } }
{ x = { x = { x = { x = {} } } } }

Object with a lot of fields.

{ zero = {}
, one = { x = {} }
, two = { x = { x = {} } }
, three = { x = { x = { x = {} } } }
, four = { x = { x = { x = { x = {} } } } }
}

Recursive object. The $ character represents the current object.

{ x = $ }

Field access.

{ x = {} }.x

Merging two objects. Merging is denoted by juxtaposition instead of an explicit operator.

{ x = {} } & { y = {} }
{ x = {}, x = y } & { y = z }
